## The three principles we take away

1. **Context is the capability.** Schema names, Skill descriptions, agent context — the engineering work that multiplies everything else.

2. **The architecture is the interface.** MCP servers, Skill registries, governance — not admin overhead. The product.

3. **Capability compounds with ownership.** A guild with no connector decays. A connector with no guild has no leverage.
